# Workspace Primitives

Folder structure, account management, auth recovery. Load when setting up a new operation, switching accounts, or recovering from auth failure.

## Cross-Project Layout

This skill is **globally installed** — it loads from any project on this device, not just `D:\NotebookLM\`. The skill itself, the automation hooks, and the CLI auth all live outside the workspace:

| Component | Location | Why there |
|-----------|----------|-----------|
| Skill files (SKILL.md, references/) | `C:\Users\user2\.claude\skills\notebooklm-workspace\` | Global skills dir → loads for any project |
| PostToolUse hooks registration | `C:\Users\user2\.claude\settings.json` | Global hooks → fire from any project |
| Hook dispatcher script | `D:\NotebookLM\.claude\hooks\post_tool_use.py` | Self-anchors to workspace via `__file__`; the registration just points here |
| Sync/ingest scripts | `D:\NotebookLM\scripts\` | Self-anchor via `Path(__file__).resolve().parent.parent`, safe to invoke from any cwd |
| `nlm` CLI auth tokens | User profile (not project) | Shared across all nlm CLI and MCP calls |
| `.env` (Notion tokens, NOTION_SYNC flag) | `D:\NotebookLM\.env` | Loaded via absolute path by the hook + scripts |

**Operational consequence:** always use absolute script paths (`python D:/NotebookLM/scripts/...`). Never `cd` into the workspace first — scripts don't need it, and doing so is misleading in a session rooted in a different project.

## Folder Structure (Hybrid: type-first, notebook-second)

```
D:\NotebookLM\
├── audio/{slug}/               ← downloaded audio overviews (.mp3, .m4a)
├── video/{slug}/               ← downloaded video overviews (.mp4)
├── infographics/{slug}/        ← downloaded infographics (.png, .jpg, .pdf)
├── slides/{slug}/              ← downloaded slide decks (.pdf, .pptx)
├── sources/{slug}/             ← scraped/recovered source files (.md, .txt, .pdf)
├── exports/{slug}/             ← assets staged for manual Notion upload (deprecated — use sync_notion_assets.py)
├── inbox/                      ← drop files here for ingestion
│   ├── processed/              ← originals moved here after ingestion
│   ├── queue.md                ← watcher log
│   └── pending-plan.json       ← ingest.py output
├── scripts/                    ← all sync + ingest Python scripts (self-anchoring, safe from any cwd)
├── claude-memory-compiler/     ← session hooks (not for notebook scripts)
├── .claude/                    ← workspace-local hook dispatcher only
│   ├── settings.json           ← placeholder; hook *registration* now lives in ~/.claude/settings.json
│   └── hooks/post_tool_use.py  ← the hook script itself — fires from any project
├── .env                        ← Notion tokens, API keys, NOTION_SYNC toggle
├── CLAUDE.md                   ← workspace overview
├── notebooks.md                ← notebook inventory (regenerated by sync script)
└── research-queue.md           ← local mirror of 🔬 Research Tasks DB
```

**Convention:** Before downloading any asset, ensure the slug folder exists:
```bash
mkdir -p "D:/NotebookLM/audio/{slug}"
```

The `{slug}` is the `Slug` property on the Notion Notebooks DB row — auto-derived from Name by `name_to_slug()` but customizable. Use `nlm notebook list --json` to get the full UUID if needed for source_add.

## Multi-Account Management

Each Google account is a separate profile. Currently only one profile is active: `default` (dataprivacyoffice@gmail.com).

```bash
nlm login profile list            # See all profiles + active one
nlm login switch <profile>        # Switch active account (MCP switches instantly too)
nlm login profile delete <name>   # Remove a profile
nlm login profile rename <old> <new>
```

The MCP server always uses the active default profile. CLI and MCP share the same auth backend — `nlm login` authenticates both.

## Auth Recovery

Sessions expire in ~20 minutes. When any MCP tool returns an auth error — **handle it silently, never prompt the user**.

### Three-step recipe

```bash
# Step 1 — Re-authenticate (Chrome launches headlessly if already logged in)
"/c/Users/user2/AppData/Roaming/uv/tools/notebooklm-mcp-cli/Scripts/nlm.exe" login
```

```
# Step 2 — Reload tokens into MCP server
refresh_auth()   # via mcp__notebooklm-mcp__refresh_auth
```

```
# Step 3 — Retry the original MCP call
```

### Quirks

- `nlm login` may **exit with code 1** on Windows due to a console encoding bug (can't print the ✓ checkmark) even when auth succeeds. Treat exit code 1 as success if Chrome launched and closed normally — look for "Closing Google Chrome" in stderr/stdout.
- Auth recovery should take <30s end-to-end. If it's stuck longer, the Chrome session itself has expired — the user needs to sign in via a browser window once.

## Notebook Inventory

Source of truth: Notion Notebooks DB (`NOTION_NOTEBOOKS_DS_ID` for queries, `NOTION_NOTEBOOKS_DB_ID` for creates). `notebooks.md` is a regenerated local mirror — do not hand-edit.

Refresh:
```bash
python scripts/sync_notion_notebooks.py --sync
# push: CLI → Notion (creates missing, updates mutable fields)
# pull: Notion → notebooks.md
# sync: push then pull
```

After any account switch, run `nlm notebook list --json` + the sync to refresh the inventory.
